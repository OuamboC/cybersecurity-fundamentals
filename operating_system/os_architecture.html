Non-Executable bit marks memory regions (e.g. stack) as non-executable. Prevents shellcode injected into data regions from running as code.

Permissions & ACLs
Standard Unix permissions (rwx) plus Access Control Lists give fine-grained control over who can read, write, or execute files. Supplemented by mandatory access controls via SELinux or AppArmor, which enforce policy-based restrictions beyond standard permissions.

Device Drivers & Modules
KERNEL PRIVILEGE
Kernel-Level Execution
Drivers run with full kernel privileges ‚Äî a vulnerability here can compromise the entire system. This makes drivers high-value attack targets.
Driver Signing
Modern OSes require cryptographically signed drivers. Unsigned or tampered drivers are blocked from loading, preventing malicious kernel modules.
Sandboxing & Modular Loading
Drivers are sandboxed where possible and loaded only when needed (modular), reducing the attack surface exposed at any given time.

Boot & Firmware
PRE-OS ¬∑ LOWEST LAYER
UEFI & Secure Boot
UEFI replaces legacy BIOS and initialises hardware. Secure Boot cryptographically verifies each component in the boot chain ‚Äî bootloader, kernel, drivers ‚Äî before execution. Only binaries with trusted signatures are allowed to run.
chain of trustTPMUEFI
Threats Defended
Without Secure Boot, attackers could plant bootkits or firmware-level rootkits that load before the OS and persist invisibly, surviving even full OS reinstalls. Secure Boot breaks this attack vector.

Hardware
PHYSICAL LAYER
CPU & Protection Rings
CPU enforces privilege rings (Ring 0‚Äì3). Kernel runs in Ring 0 with full access; user apps in Ring 3 with restricted access.
Physical Memory (RAM)
The hardware foundation for all virtual memory abstractions. MMU (Memory Management Unit) performs address translation enforced by hardware.
Storage & Peripherals
Physical disks, network cards, USB controllers. All access must be mediated by the OS via device drivers ‚Äî never directly from user space.
